## 1. 개요

본 프로젝트는 서울시 상권분석서비스 등 **공공 상권 데이터**를 활용하여, 예비 창업자가 입력한 조건에 맞는 **창업 입지(상권)** 를 추천하는 **웹 프로토타입**을 개발한다.

- **입력 예**: 희망 업종, 타겟 연령, 희망 지역, (선택) 성별·주말/평일·시간대 선호  
- **출력**: 추천 상권 **TOP 5**, 항목별 **추천 사유**, (가능 시) **지도** 표시  

데이터 분석은 추천의 근거를 마련하는 단계이며, 프로젝트 완료 기준은 **「조건 입력 → 맞춤 TOP 5 + 사유」가 동작하는 웹 서비스**이다.

## 2. 추진 배경 및 필요성

### 2.1 배경

#### (1) 문제 인식

소상공인 창업 시 가장 부담이 큰 결정은 **입지(상권)** 와 **업종** 선정이다. 유동인구, 임대료, 주변 분위기 등은 널리 쓰이지만, **목표 고객(연령·성별·이용 시간)** 과의 맞음을 숫자로 비교하기는 어렵다.

20·30대(이하 **2030**)를 주요 고객으로 하는 업종은 상권마다 패턴이 다르다. 같은 카페·음식 업종이라도 어디는 **주말 매출**이 강하고, 어디는 **평일 점심·저녁**이 강하다. 이를 **추정매출·소비 구조 데이터**로 비교할 필요가 있다.

#### (2) 기획 계기

팀원 중 한 명이 교내 **RISE CREW 창업 동아리**에서 활동하며, 소상공인·예비 창업자가 **입지 선정**과 **타겟 고객 분석** 과정에서 어려움을 겪는 사례를 직접 접하였다. 상권 정보는 많지만 “**내 업종·내 고객**에 맞는 곳”을 고르기는 여전히 감과 경험에 의존하는 경우가 많았다.

이에 우리는 공공데이터 기반의 **상권·소비 패턴 분석**으로 위 과정을 보완하고, 창업자가 조건을 입력하면 **맞춤형 입지 후보**를 제시할 수 있는 서비스를 기획하게 되었다.

### 2.2 필요성

| 구분 | 기존 | 본 프로젝트 |
|------|------|-------------|
| 정보 | 상권 통계·지도 조회 | **사용자 조건 반영** 맞춤 추천 |
| 근거 | 유동인구·정성 판단 | 추정매출·점포·(선택) 인구 등 |
| 결과 | 일반 정보 | **TOP 5 + 추천 사유** (의사결정 보조) |

## 3. 목표 및 산출물

### 3.1 목표

1. 상권·업종별 **2030 소비 비중**, 요일·시간대 패턴을 분석한다.  
2. 생활밀접 업종 **62종 전체**를 선택 가능하게 하고, 조건별 상권 **적합도**를 비교한다.  
   - *2025년 데이터 기준 62종이었으며, 추후 조사·갱신에 따라 변동될 수 있음.*  
3. 웹 프로토타입으로 **맞춤형 입지 TOP 5**와 **추천 근거**를 제공한다.

### 3.2 산출물 및 완료 기준

| 구분 | 내용 |
|------|------|
| 핵심 산출 | 웹 프로토타입 (입력 → TOP 5 → 사유 → 지도) |
| 내부 | EDA, 지표·가중치 정의 |
| 학회 | 발표 자료, 시연 시나리오 |

**완료 기준**

- 업종 **62종 중 임의 1종 이상** 추천 동작  
- **TOP 5** 및 **추천 사유** 출력  
- MVP 데이터(§6.1) 반영 및 **상권 지도** 표시  
- 결과 화면에 **데이터 기준 분기** 및 참고용 안내 표기

## 4. 서비스 개요

### 4.1 정의

**개인 맞춤형 창업 입지 추천 서비스** — 예비 창업자 조건에 맞는 상권 5곳을 순위·근거와 함께 제시한다.

### 4.2 입력 (추후 변경 가능)

| 항목 | 비고 |
|------|------|
| 희망 업종 | 서비스 업종 **62종 전체** |
| 타겟 연령 | 10~60+, 2030(20+30) 등 |
| 희망 지역 | 자치구, 상권명, 상권 유형(골목·발달·전통·관광) |
| 타겟 성별 | 남·여·무관 |
| 운영 패턴 | 주말형 / 평일형 / 점심(11~14) / 저녁(17~21) |

### 4.3 출력

| 항목 | 내용 |
|------|------|
| 추천 목록 | 적합도 순 **TOP 5** |
| 추천 사유 | 2030·매출·시간대·경쟁 등 2~3문장 |
| 부가 | 상권 유형, 분기 매출 규모, 지도 하이라이트 |
| 고지 | 기준 분기, 추정·참고용 안내 |

### 4.4 처리 흐름

```
[조건 입력] → [후보 필터] → [적합도 점수] → [TOP 5·사유] → [결과·지도]
```

## 5. 추천 방식

### 5.1 방식 후보

- **가중치 합산 방식** (필터 + 부분 점수 + TOP 5 + 사유 문장)  
- **상권 클러스터링** (주말형·2030형 등 유형 라벨·사유 보강)  

※ EDA 및 시연 준비 일정에 따라 주 방식 확정. 1차 구현은 **가중치 합산**을 기준으로 진행.

### 5.2 처리 단계 (가중치 합산 방식 기준)

**① 필터**

- 선택 **업종** 일치  
- **희망 지역** (자치구·상권명·유형)  
- 극소 매출 상권 제외 (EDA 후 하한 확정, 예: 분기 5천만 원 미만)

**② 부분 점수** (최신 분기·상권·업종 기준)

| 요소 | 내용 | 가중치(안) |
|------|------|------------|
| 타겟 연령 | 2030 또는 선택 연령 매출 비중 | 30% |
| 업종 수요 | 해당 업종 분기 매출 (percentile) | 25% |
| 시간·요일 | 주말·점심·저녁과 입력 일치도 | 20% |
| 성별 | (입력 시) 성별 매출 비중 | 10% |
| 경쟁·효율 | 점포 수 역반영, 점포당 매출 | 15% |

**③ 종합** — `총점 = Σ(부분점수 × 가중치)` → **TOP 5**

**④ 사유** — 상위 2~3 요소를 템플릿 문장으로 출력  
(예: 「2030 비중 48%, 서울 평균 대비 높음」)

가중치·하한은 **§9 EDA** 후 6월 초 확정.

## 7. 분석 설계

*초안이며, EDA 결과에 따라 조정할 수 있다.*

### 7.1 파생 지표

| 지표 | 산출 |
|------|------|
| 2030 비중 | (연령 20+30 매출) / 분기 매출 |
| 주말 비중 | 주말 / (주중+주말) |
| 점심·저녁 비중 | 11~14, 17~21 / 시간대 합 |
| 객단가 | 분기 매출 / 분기 건수 |
| 점포당 매출 | 분기 매출 / 운영 점포 수 |

### 7.2 분석 항목

1. 업종·상권별 2030·매출·객단가  
2. 요일·시간·주말 패턴  
3. 점포 수·점포당 매출 (경쟁)  
4. (선택) 2019~2025 분기·연도 추이  
5. (선택) 상권 유형화 → 주말형·2030형 라벨  

### 7.3 추정매출로 가능한 분석

- 2030 매출이 몰리는 상권·업종  
- 주말형 / 평일형 상권 구분  
- 시간대별 인기 상권  
- 성별·연령 타겟 상권  
- 분기별 변화  

→ **「어디서, 무슨 업종이, 누가, 언제」** 소비되는지 파악  

### 7.4 점포 데이터로 보강

- **경쟁 강도**: 동일 업종 점포 수  
- **점포당 매출**: 시장 대비 과밀 여부  
- 매출만으로 안 보이는 리스크 (매출↑·점포↓ 등)

## 11. 제약 사항 및 향후 과제

### 11.1 제약

- 추정매출 = **상권·업종 집계 추정치** (개별 가게 실매출 아님)  
- **분기 단위** 갱신, 분기 종료 후 약 2개월 지연  
- 연령·성별 합 ≠ 총매출 가능  
- 임대·권리금·순이익 미포함 → **후보 축소·참고** 수준  

### 11.2 향후 과제

| 과제 | 내용 |
|------|------|
| **실시간·준실시간** | 공개 상권 매출 실시간 API 없음. 최신 분기 자동 반영·본인 가게 매출 API(매통조 등) 연계 검토 |
